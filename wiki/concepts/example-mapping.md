# Example Mapping

## Definition
Example Mapping is a collaborative discovery technique used in Behaviour-Driven Development (BDD) workshops. It provides a structured conversation format for breaking down user stories into a set of rules and concrete examples, typically using index cards (or digital equivalents) during a facilitated meeting.

## Key Characteristics
- **Structured conversation format**: The technique guides a team through a facilitated discussion to extract acceptance criteria and examples.
- **Breakdown of stories**: User stories are dissected into explicit rules (business rules) and examples that illustrate those rules.
- **Index cards or digital boards**: The method often uses physical index cards or a digital equivalent to represent rules, examples, and questions.
- **Collaborative and discovery-focused**: Example Mapping is designed to be done as a team, usually involving developers, testers, and domain experts.
- **Directly supports BDD discovery**: It helps teams decide precisely what to build before writing code, aligning with the BDD "discovery" practice.
- **Complementary to other workshop techniques**: Mentioned alongside Event Storming and Specification by Example as a pattern that complements BDD practices.

## Applications
- **Behaviour-Driven Development discovery workshops**: Example Mapping is used during the BDD discovery phase to clarify requirements and define precise examples before automating them.
- **Requirement refinement**: The technique helps teams break down vague or high-level stories into testable, concrete acceptance criteria.
- **Agile planning and backlog grooming**: Useful for refining user stories with the entire team, reducing ambiguity and ensuring shared understanding.
- **Any scenario where rules and examples need explicit documentation**: The card-based format makes it easy to capture, organize, and visualise the logic behind a story.
